Get Git and Basic UNIX Commands

09 September 2014

Git is a version control system that allows you to easily maintain your code and collaborate with others.

If you're new to software development, the idea of version control may be a little foreign.

On the surface, version control gives you the opportunity to save your code, and view those saves over time.

Let's work through building a simple website using Git to commit our code online.

First off you should have Git installed. To check if you have a version of Git. To do this, simply open your terminal and type the following command:

$ git --version

If Git is installed your terminal will tell you the version of Git installed on your computer. If it's not installed you will see an error message.

If you don't have git installed follow the installation guide at: http://git-scm.com/book/en/Getting-Started-Installing-Git

We will also be using Sublime Text which you can download for free here: http://www.sublimetext.com/3

1. Create Files

First off we want to write a small bit of code so that we can begin using Git.

We'll build out our website through our terminal using basic UNIX commands.

Open up the terminal and type:

$ cd

This command stands for Change Directory. This is how we move around and through files. Typing this command all by itself directs your terminal to the root directory of your computer.

Let's get to our Desktop by typing:
$ cd Desktop

and let's make a new folder for our project:

$ mkdir my-git-project

The command mkdir stands for Make Directory or in other words a folder. We also named the directory "my-git-project".

Let's enter the directory:

$ cd my-git-project

Now we want to to create the files of our project.

$ touch index.html
$ touch main.css

The touch command allows us to create a new file. Index.html is a special file that tells our browser that it's the main hub of our code. main.css is where we will store our styles.

2. Write some code

We should have sublime text installed, so let's go ahead and open our project.

You can look through Finder to spot the project files and then open them with Sublime Text. Alternatively I like to open sublime from the terminal. To do this you'll first need a link, create a link by copying this in your terminal:

ln -s "/Applications/SublimeText.app/Contents/SharedSupport/bin/subl" /usr/local/bin/sublime

If that worked, you'll be able to open sublime text like this:

$ sublime .

Now that we have our files open in Sublime Text let's write some code!

Write a basic html page:
index.html 

<!DOCTYPE html>  
<html>  
<head>  
    <title>GetGit!</title>
    <link href="main.css" rel='stylesheet' type='text/css'>
</head>

<body>  
    <div class="container">
        <h1>Hello World!</h1>
    </div>
</body>

Let's open our main.css file and write some styles:

.container {
    background: #D6D6D6;
    font-family: sans-serif;

Now back to our terminal to view our website. In the terminal type the command:

$ open index.html

This file should open your web browser with our index.html file open. When that happens it should look like a gray box with the words Hello World.

3. Time for Git

Now that we've created a project and have written some code, let's add git to it all.

First we'll initialize the app by typing in our terminal:

$ git init

Then we will check the status:

$ git status

and we should see: 

On branch master

 Initial commit

 Untracked files:
   (use "git add <file>..." to include in what will be committed)

    index.html #this should be red
    main.css #this should be red

nothing added to commit but untracked files present (use "git add" to track)

The red files are the ones we need to add in order for git to track their changes. Do this by typing:

$ git add index.html
and
$ git add main.css

Now if we run:

$ git status

We get:

 On branch master

 Initial commit

 Changes to be committed:
   (use "git rm --cached <file>..." to unstage)

    new file:   index.html #GREEN
    new file:   style.css #GREEN

Now we are ready to commit (or save) our code to git! We do this by typing:

$ git commit -m "first commit"

This is how we save code with Git. Now as we contiune working on our code we go through the same git commands every time we want to commit:

$ git status
$ git add [NAMES OF RED FILES]
$ git commit -m "WHAT YOU WORKED ON"

4. Bonus round

A popular tool used while writing code with Git is called GitHub. GitHub is a web application that allows you to store your code files online.

Lucky for us we are able to quickly integrate GitHub with our current website.

First you'll need a GitHub.com account. Sign up at: https://github.com/

Once you've signed up, create a new Repository, or repo, on github. You should be able to easily find a green button on their website that allows you to make a new repo.

We're going to call our repo my-git-project.

Once you create a project, github has some good instructions on how to hook it up to our project, but for today, let's just hook our remote to our project. The remote allows us to push our project from our computer to GitHub.

To do this type in the terminal:

$git remote add origin https://github.com/[USERNAME]/my-git-project.git

The [USERNAME] should be the username you created when you made your GitHub profile.

Now we are ready to push! Type the following into the terminal:

$ git push -u origin master

Now if we look on github we should be able to find our code!

RECAP

Now as you work on your project, you can do the "GitHub Dance":

$ git status
$ git add [NAMES OF RED FILES]
$ git commit -m "WHAT YOU WORKED ON"
$ git push
